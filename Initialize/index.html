前言

Hexo本身的簡潔有力能夠讓使用者在看文章的過程中感到舒適，藉由學習一項新的技術同時也來更新一下這幾年所遇到的專案問題，也能夠將當初的寫部落格初衷繼續傳承下去，取之於網路，回饋於網路。

為什麼想要重建一個部落格?

主要原因

Hexo相當簡潔有力

自2018年開始使用了Google提供的Blogger平台作為簡單紀錄開發內容的部落格之後，就一直沒有對部落格的外觀問題好好處理，反而只針對專案內容傾心研究，過程中所研究的內容也是糊里糊塗的就丟上去平台分享，一直到我發現了Hexo(也就是您目前所見的這個網頁)，才知道原來我之前的Blogger真的有夠醜，Hexo所製作的內容相當的簡潔有力!

額外好處

網站自主權，非常的重要!

Hexo作為一個靜態網頁框架，可以在本地端透過套件進行靜態網頁渲染，從而讓使用者可以自由指定要Host的地方，而剛好GitHubPage也有提供這樣的服務，讓網頁的內容可以藉由佈署在Github進行呈現，透過自己網域進行CNAME設定即可有自己的官方網站部落格了，不用擔心有一天Blogger服務被Google收掉的話，所建立的內容會血本無歸。

未來文章組成方向

撰寫有價值的文章!

開始撰寫部落格的初衷也是因為有一群大神願意在網路世界的某處將所習得的知識分享給各位，期許我也能夠成為像是這樣有用的人，撰寫出一些能夠對這個世界有所幫助的文章。

這幾年來與各地網友交流，也大概可以理解我能夠幫助到那些部分，尤其是對於VBA在工程上的設計監造應用專案最為受各位所喜愛，在此，我也很感謝曾經幫助過我的朋友，因為有你們的支持，也才能讓我有動力繼續研究開發，未來文章發展方向會由以下三點進行

程式寫作基本原理

  • Excel VBA

    儲存格、工作表、活頁簿的屬性操作，表單的設計流程，類資料庫的編排，報表的呈現方式…

  • AutoCAD VBA (ProgeCAD VBA)

    圖元操作(點、線、面、文字)、配置設定、與外界數據交互…

  • Google Apps Script

    Google Sheet、Gmail、Drive、WebApp網頁表單、WebAPI與外界資料交換

  • LINE

    LINE Notify通知、LINE BOT機器人(結合Flask、GAS的WebAPI)

  • Python

    藉由引用Python相關套件做成的套件提供給Excel無法達成的功能作為使用、以及最近開始接觸的Streamlit，這是一個不需要理解前端介面的好工具，可以很快的發展一個線上服務。

專案開發的過程紀錄

目前所採用的語言為了加速開發的時間，大多數都是沒有很嚴謹的程式語言，也就是不會針對程式最基礎的部分進行琢磨(包含資料型式的比較、互動)，所以會遇到不如預期的內容也算是滿正常的，能夠透過採坑的過程中將可能會預見的提前告訴世人，也是功德一件，當然這些內容都可以透過單元測試的過程作避免的手段，只不過有更多的人只是希望東西能夠動起來最要緊，也會變成技術債…

專案執行的可能發展方向

專案執行會有一些朋友參與，不同的使用者有各式各樣的需求，這部分對我而言都是很好的回饋，如果這些需求能夠讓程式更適用於大多數人的需求，這樣對於專案執行的發展方向就會是健康的，畢竟東西好壞各有所主觀意識，但若能夠大部分的人都認為這樣很棒，那我也很樂意為之，所以遇到問題的時候請用力告訴我，這樣我會很開心!
